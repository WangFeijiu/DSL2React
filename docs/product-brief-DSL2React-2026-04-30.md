# Product Brief: DSL2React

**Date:** 2026-04-30
**Author:** Project Owner
**Version:** 1.0
**Project Type:** library
**Project Level:** 2

---

## Executive Summary

DSL2React 是一个从设计到代码的自动化转换工具，将 MasterGo 设计稿（通过 DSL.json）转换为高还原度的 HTML 和 React 代码。主要服务于需要快速开发大量营销落地页的前端开发团队。该项目旨在将落地页开发时间从 1-2 天缩短到 2-3 小时，支持即将到来的流量投放和 SEO 需求。

---

## Problem Statement

### The Problem

目前团队使用 MasterGo 设计营销页面和落地页面，但从设计稿到代码的转换完全依赖手工编写。由于需要支持响应式布局且页面相对复杂，开发一个落地页面普遍需要 1-2 天时间。

**具体例子**：设计师在 MasterGo 中完成一个营销落地页设计后，前端工程师需要：
1. 仔细梳理设计稿的布局、样式、动画
2. 手工编写 HTML/CSS/React 代码
3. 反复调试以达到设计稿的还原度
4. 处理多种屏幕尺寸的响应式适配

这个过程耗时且重复性高。

### Why Now?

项目即将开始大规模投放流量，需要大量静态化页面来支持 SEO 优化。现有的手工开发方式无法满足快速产出的需求，必须找到自动化解决方案来大幅提升开发效率。

### Impact if Unsolved

如果继续使用现有方式：
1. **开发周期过长**：无法快速响应业务需求，错失流量投放窗口期
2. **人力成本高**：需要投入大量前端资源在重复性工作上
3. **代码质量不一致**：手工编写导致不同页面的代码风格和质量参差不齐

## Solution Overview

### Proposed Solution

DSL2React 提供一个自动化的设计到代码转换流程：

**核心流程**：
```
MasterGo 设计稿 (通过 MCP)
    ↓
DSL.json 文件
    ↓
生成 HTML 文件 (MVP 重点)
    ↓
浏览器插件微调 HTML (未来版本)
    ↓
生成 React 代码 (未来版本)
    ↓
在 React 项目中预览
    ↓
集成外部组件库，生成可复用代码
```

**MVP 阶段**专注于验证 DSL → HTML 转换的可行性和还原度，这是整个项目的技术基石。

### Key Features

**MVP 阶段（DSL → HTML）**：
- DSL.json 到 HTML 的高还原度转换（目标 85%+）
- 支持响应式布局（多种屏幕尺寸适配）
- 支持动��效果
- 支持 MasterGo 的主要设计元素（布局、样式、文本、图片、图标等）

**未来版本**：
- 浏览器插件可视化微调
- HTML → React 代码生成
- 外部组件库集成
- 复杂交互支持
- MasterGo MCP 集成

### Value Proposition

**将落地页开发时间从 1-2 天缩短到 2-3 小时**，实现 80-90% 的效率提升，让前端工程师从重复性的设计还原工作中解放出来，专注于更有价值的业务逻辑开发。

## Constraints and Assumptions

### Constraints

- **技术约束**：HTML 还原度必须达到 85%，否则项目无继续价值
- **时间约束**：MVP 必须在 1 个月内完成验证
- **浏览器约束**：优先支持 Chrome，暂不考虑其他浏览器
- **功能约束**：MVP 阶段不包含复杂交互、插件、埋点等功能

### Assumptions

- MasterGo 可以导出标准的 DSL.json 格式
- 前端团队熟悉 React 和组件库开发
- 现有基础设施可以支持页面预览和部署
- 使用成熟的技术栈可以降低开发风险
- 85% 的还原度足以满足业务需求

## Risks and Mitigation

### 技术风险（高）

**风险**：HTML 还原度无法达到 85% 的目标
- **可能性**：中等（已尝试过多个工具都未达标）
- **影响**：高（项目核心价值无法实现）
- **缓解策略**：
  - MVP 阶段专注验证这一核心能力
  - 快速迭代，及时调整技术方案
  - 与设计团队紧密合作，明确还原度评估标准
  - 考虑引入 AI 辅助提升还原度

### MasterGo DSL 格式变化风险（低）

**风险**：MasterGo 更新导致 DSL 格式变化
- **可能性**：低
- **影响**：中等（需要适配新格式）
- **缓解策略**：
  - 设计灵活的解析器架构
  - 与 MasterGo 保持沟通
  - 建立版本兼容机制
